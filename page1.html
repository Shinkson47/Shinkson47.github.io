Planned Features

What's planned for implementation

  1. Thread manager - Create, monitor, and manipulate threads with less code.
  2. Error manager - Handles problems you've may not have even thought of, with the goal purpose of keeping your game stable and running. This includes exception handling for all threads, as well as warning and logging capabilities. It's most prominent feature is it's capability to detect error cascades.
  3. Input Management - Map Key and Mouse inputs to any call you like, with the ability to swap between presets on the fly, and assign different key configs to each content window. I may also include controller support in the future.
  4. Internal Executables - Classes which extend the Java Runnable class to execute within the JGEL runtime environment. These will automatically be executed in JGEL managed threads, with unified error handling.
  5. Window Management - Create and manage swing windows with less code. Assign jgel content windows, and use a graphics library of your choice.
  6. Graphics implementations - The implementation of exsisting graphics librarys provides the foundation of a well built industry for developing great games. JGEL will implement some graphics features for menus and the like, but will leave the heavy lifting to the well routed big boys.
  7. Documentation - Currently yet to be published, many markdowns explaining classes, and how to get started are currently being written.
  8. Console -  A simple, practacle and expandable console interpreter, accessable through the runtime terminal, and eventually through debugging GUI and ingame, similar to what can be achieved with the Creation and Source engines.
  9. OPEX Visual - Visual development tools and runtime manager

17 January 2020

Version 1 Dropped

JGEL V1 Was pushed for the final time. For many reasons, but mostly because the project has little pre-thought, and was becomming more unorganised and unprofessional by the commit. I have plans for how i want JGEL to function, and be interacted with, and this implementation is not addequate. However it paved the way for multiple systems, and provide a good insight to what this project would be like.

31 March 2020

JGEL Webpage was born

During a much needed revamp to shinkson47.in, the very page you're looking at right now was added.

3 April 2020

Console Interpreter

Developed the framework basis for the engine's console interface, accessable via the runtime terminal. It uses classes for instructions, and is entirely expandable.

5 April 2020

Theorised Instruction Builder

After writing multiple instruction classes by hand, and realising how they're all so alike, the idea of an instruction builder tool was born.

10 April 2020

Created Instruction Builder

Created a basic GUI tool for the JGEL toolbox, providing a simple way to generate and regenerate console instruction classes.

12 April 2020

Preliminary Instruction Builder Testing

Began using the tool to rewrite, and replace exsisting hand written instructions to test functionality. Also employed the help of a third party to test the tool for robustness.

14 April 2020

Begginging of JavaPoet addoption

The intruction builder's getting a revamp! Whilst functional, the importation of javapoet will slimline and generally improve the structure of java class generation witin the console builder.

8 May 2020

Project refactorisation

Project JGEL was finally given a dedicatated name, Phoenix, leading to the refactorisation of the project to "Open Phoenix Engine", or OPEX.
